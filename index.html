My Skills

HTML

Over the years my knowledge of HTML as passed from XHTML to HTML 4.0 to HTML 5.

Always exploring the new features, APIs and tricks to make the most out of the web.

The new features of HTML5 which I'm more excited about include WebSockets, LocalStorage or AppCache.

CSS

It's an exciting time to play with CSS, nowadays we can accomplish things that yesterday required tons of JavaScript to work, the gradual replacement of layout images in favor of pure CSS techniques is one of the true goals of the front-end developers.

One of my favorite features of the new CSS is playing with the 3D effects and perspective and the :before and :after pseudoelements

The possibilities extend further when we use CSS preprocess tools such as Sass or Less

Responsive Design

Mobile browsing has grown exponentially on the last months, what begun as a trend has become a necessity and nowadays I try to build responsive and/or fluid websites that can have different layouts depending on the screen viewing it.

Because it's not just mobile or desktop anymore since the mobile screens have diversified so much lately modern layouts have to be able to adapt to any screen size whatsoever.

To accomplish this I lean on CSS grid frameworks and media-queries to achieve a highly malleable layout

Server Administration

Knowing how to handle, provision and manage servers it's a very important skill on web development, for this reason I try to stay up to date to the most efficient way to handle this processes.

Using Composer is one of the best ways to handle external libraries and dependencies of your project.

Other tools like Vagrant or Puppet kind be very handy too, specially if you have to handle and provision several instances or servers at the same time.

User Experience / User Interface

This had become a major area of influence of my work. Being able to affect the concept of a website before and after the wireframes and design its crucial to build an interface that is both comfortable and engaging to the user.

I think that the every layout decision of a website has to have a reason to be, and it can't only be an artistic one.

For these reasons I like and enjoy meeting and discussing with designers the best approach at it design decisions and the technical limitations that might interfere.

Latest Work

http://bestivalbuzz.recognitionlondon.com/

Bestival Buzz Tracker

Build with the d3.js library each band was represented as a circle or bubble. The radius of its bubble was tied with the ‘buzz score’ of the band which varied depending on how much people were talking about that band on twitter, there were also different filters you can apply to the bubbles to change the point of gravity and separate them depending on the genre of music, the top 5 bands or even transform the bubbles into points of a dynamic chart where you can compare factors like tweets, mentions, followers or hashtags, all of these updating with real time tweets.

Latest Experiments

http://recme.meteor.com/

RecMe (Recommend me places)

It's an application build with Meteor, I created it when I first got to London, so people could recommend me interesting places to go, to eat, to go out, etc.

It was also a perfect time to test how fast rapid development can go with Meteor, the whole application was done in one afternon and with only 194 lines of Javascript for both the front end and the back end, well to be fair Meteor is so cool I didn't event need a single line on the backend for it to work
